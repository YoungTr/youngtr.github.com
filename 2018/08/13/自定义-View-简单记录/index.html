自定义 View 查漏补缺

发表于 2018-08-13 | 分类于 Android自定义View

1. Path.setFillType(Path.FillType ft) 设置填充方式

  • EVEN_ODD
  • WINDING (默认值)
  • INVERSE_EVEN_ODD
  • INVERSE_WINDING

其中后面的两个带有 INVERSE_ 前缀的，只是前两个的反色版本

参考：even-odd rule （奇偶原则）、non-zero winding rule （非零环绕数原则）

2. Paint 颜色 - Shader

Paint.setShader(Shader shader)

  • LinearGradient
  • RadialGradient
  • SweepGradient
  • BitmapShader
  • ComposeShader

BitmapShader

如果想绘制圆形的 Bitmap，就别用 drawBitmap() 了，改用 drawCircle() + BitmapShader 就可以了（其他形状同理）。

  • mode: 两个 Shader 的叠加模式，即 shaderA 和 shaderB 应该怎样共同绘制。它的类型是 PorterDuff.Mode 。

参考：自定义 View 1-2 Paint 详解

https://github.com/hdodenhof/CircleImageView

自定义圆形头像CircleImageView的使用和源码分析

各个击破搞明白PorterDuff.Mode

3. setStrokeCap(Paint.Cap cap)

设置线头形状

  • BUTT (默认)
  • ROUNT
  • SQUARE

4. setStrokeJoin(Paint.Join join)

设置拐角形状

  • MITER (默认)
  • BEVEL
  • ROUNE

5. setStrokeMiter(float miter)

当线条拐角为 MITER 时，拐角处的外缘需要使用延长线来补偿

6. setFilterBitmap(boolean filter)

设置是否使用双线性过滤来绘制 Bitmap

7. setPathEffect(PathEffect effect)

  • CornerPathEffect 把所有拐角变成圆角

  • DiscretePathEffect 把线条进行随机的偏离，让轮廓变得乱七八糟。乱七八糟的方式和程度由参数决定。

  • DashPathEffect 使用虚线来绘制线条

  • PathDashPathEffect 这个方法比 DashPathEffect 多一个前缀 Path ，所以顾名思义，它是使用一个 Path 来绘制「虚线」

  • SumPathEffect 这是一个组合效果类的 PathEffect 。它的行为特别简单，就是分别按照两种 PathEffect 分别对目标进行绘制

  • ComposePathEffect 这也是一个组合效果类的 PathEffect 。不过它是先对目标 Path 使用一个 PathEffect，然后再对这个改变后的 Path 使用另一个 PathEffect
